API Documentation & Import

Download the OpenAPI specification to import into HCL AppScan or other API security scanning tools.

How to use with HCL AppScan:

  1. Download the OpenAPI specification file
  2. Open HCL AppScan (AppScan on Cloud or AppScan Enterprise)
  3. Create new scan → Select "Import API Definition"
  4. Choose "OpenAPI 3.0" format
  5. Upload the downloaded JSON file
  6. Configure login credentials (use test account provided by Simple Bank administrator)
  7. Configure MFA if testing with 2FA enabled
  8. Run DAST scan

💡 Known Vulnerabilities Included:

  • SQL Injection (GET /api/search?q=)
  • IDOR - Insecure Direct Object References (transfers, transactions)
  • Stored XSS (POST /api/profile/comment)
  • CSRF (POST /api/transfer)
  • Broken Authentication (MFA bypass tests)
  • Privilege Escalation (admin panel access)

Use these endpoints to test the API.

🔑 Getting Started: How to Get Your Session ID
  1. Step 1: Log in using the login endpoint below
  2. Step 2: Your browser/client automatically stores the session cookie
  3. Step 3: When using curl, save the cookie with -c cookies.txt during login
  4. Step 4: Use the saved cookie for all other requests with -b cookies.txt
  5. Step 5: If using Postman, enable "Automatically follow redirects" and cookies will be saved in the Environment
POST /api/auth/login
Login with email & password. Returns user info and sets session cookie.
curl -X POST https://appscan-demo-mfa-production.up.railway.app/api/auth/login \
-H "Content-Type: application/json" \
-c cookies.txt \
-d '{"email":"[test-account-email]","password":"[test-account-password]"}'
Note: If user has MFA enabled, add "totp_code":"123456" to the request
GET /api/user/profile
Get current user's profile information
curl -X GET https://appscan-demo-mfa-production.up.railway.app/api/user/profile \
-b cookies.txt
GET /api/user/account
Get all accounts for current user (checking and savings)
curl -X GET https://appscan-demo-mfa-production.up.railway.app/api/user/account \
-b cookies.txt
GET /api/v1/transactions
Get all transactions for current user
curl -X GET https://appscan-demo-mfa-production.up.railway.app/api/v1/transactions \
-b cookies.txt
GET /api/search?q=EMAIL_OR_NAME
Search for users by email or name
curl -X GET "https://appscan-demo-mfa-production.up.railway.app/api/search?q=demo-user" \
-b cookies.txt
POST /api/transfer
Transfer money between accounts
curl -X POST https://appscan-demo-mfa-production.up.railway.app/api/transfer \
-H "Content-Type: application/json" \
-b cookies.txt \
-d '{"fromAccountId":"ACCOUNT_ID","toAccountId":"ACCOUNT_ID","amount":100}'
GET /api/comments
Get all community chat comments
curl -X GET https://appscan-demo-mfa-production.up.railway.app/api/comments \
-b cookies.txt
POST /api/profile/comment
Post a comment to community chat
curl -X POST https://appscan-demo-mfa-production.up.railway.app/api/profile/comment \
-H "Content-Type: application/json" \
-b cookies.txt \
-d '{"content":"Your comment here"}'
POST /api/user/reserve
Reserve your account for 24 hours (demo mode)
curl -X POST https://appscan-demo-mfa-production.up.railway.app/api/user/reserve \
-b cookies.txt
POST /api/mfa/enable
Enable two-factor authentication and get QR code
curl -X POST https://appscan-demo-mfa-production.up.railway.app/api/mfa/enable \
-b cookies.txt
POST /api/mfa/verify
Verify TOTP code to enable 2FA
curl -X POST https://appscan-demo-mfa-production.up.railway.app/api/mfa/verify \
-H "Content-Type: application/json" \
-b cookies.txt \
-d '{"totp_code":"123456"}'
GET /api/auth/logout
Logout and destroy session
curl -X GET https://appscan-demo-mfa-production.up.railway.app/api/auth/logout \
-b cookies.txt

💡 Pro Tips:

  • Curl: Use -c cookies.txt to save cookies on login, -b cookies.txt to use them on subsequent requests
  • Postman: Set up an Environment variable and use the login request to automatically populate it for other requests
  • Test Accounts: Contact Simple Bank administrator for test account credentials
  • Verbose Curl: Add -v flag to see all headers and cookies
  • OpenAPI Spec: Download from Settings or fetch: curl https://appscan-demo-mfa-production.up.railway.app/api/openapi.json
